Partiendo del fichero xml, desarrollar un esquema xsd con las siguientes especificaciones:

El nodo principal es cv, este nodo tiene un atributo obligatorio que se llama lang.

Dentro de cv tenemos los siguientes nodos:

- photo, name, address, email y tlf que son de tipo string
- summary que es de tipo string y es opcional
- educations que es opcional y puede aparecer como máximo 3 veces
- experiences, languages y skills que son opcionales.


Dentro de educations podemos tener un número indeterminado de nodos education, el nodo educations tiene un atributo obligatorio que se llama type de tipo string cuyos valores sólo pueden ser academic, complementary y other.

Dentro de education tenemos los nodos course e institution, que son de tipo string y pueden aparecer en cualquier orden. El nodo education tiene dos atributos, start y end que son de tipo string.

Dentro de experiences tenemos un número indeterminado de elementos experience.

Dentro de experience tenemos el elemento company que es de tipo string y el elemento projects.

Dentro del elemento projects tenemos el elemento project, que puede aparecer un número indeterminado de veces.

El elemento project tiene 4 atributos obligatorios de tipo string, title, workplace, start y end. Este elemento tiene dentro un elemento de tipo string que se llama description.

Dentro de languages tenemos el elemento language de tipo string que puede aparecer un número indeterminado de veces.

El elemento language tiene un atributo de tipo string obligatirio que se llama level.

El elemento skills tiene dentro un número indeterminado de elementos de tipo skill, que son de tipo string.
